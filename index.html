FIXING BROKEN WINDOWS

@wookiehangover

March 3, 2013

Stopping criminal behavior in your apps

Because you don't want to end up here

Everything is going great.


Until it all goes to hell.

Things that will definitely happen:

  • You will learn new techniques
  • Adopt new libraries
  • Lose & gain team members
  • Probably be responsible for more than just JavaScript

Which means breaking shit
A lot.

Broken Windows: The outward appearance of criminal behavior can lead to actual criminality

If you fix broken windows (or keep them from breaking in the first place) you can fight back against entropy

And you can also say you fight crime! Kinda like Batman.

What constitutes crime in a software project?

  • when it effects producivity
  • or creates difficult bugs
  • or boxes you in
  • or confuses you later

If you disagree with any of these, good! At least that means you're thinking critically about these problems

Like any criminal justice system, there are minor crimes

(like shoplifting)

and major offenses

(like regicide)

1 - Mixed spaces & tabs, trailing whitespace

Minor crime, major implications about process

You should care about your files

If you don't, make sure someone else does

Don't be that guy.

2 - Not having / following style guide

  • rwaldron's idiomatic.js
  • npm styleguide / anything ASI
    @fat "I know how semi-colon rules work"

But what if my personal style is good enough? I don't need someone else's rules.

Knowing the language

Knowing the browser

At a minimum, enforce code quality with JShint

3 - Know thy version control system

Minor cime, medium to long-term agony

Github makes things really easy—but simple workflows have a hard time scaling to larger teams

Merging, Rebasing, Feature Branches, Pull Requests

If you're using Git(hub), you should know what all of those things mean

3.5 - Doing it live

4 - Avoid Monolithic files

Multi-thousand line files increase cognitive load, decrease productivity

If you're generating these with a build process, its ok

Embrace Laziness

Having huge files isn't very lazy, guys

5 - Modules!

Let's have an argument!

JavaScript will have modules... eventually

Until ES6 is ratified and you can use it everywhere, you really have 2 choices

CommonJS

You need a build process for development

...But at least you're organized

Require.js / AMD

Yes, require.js is not the only flavor of AMD

But its possibly the most relevant

If you're not using modules...

At least stop abusing globals

Creating lots and lots of global variables is generally considered a "bad thing"

6 - Leave $.fn alone unless you really mean it.

Is something a good jQuery plugin?

Questions you should axe before making a plugin

  • Does it make sense for this be DOM centric? (Will it iterate over a jQuery selector)
  • Is it generic enough for other people to use?
  • Are you going to release and maintain it?

6.5 - prefer simple dependencies over complex ones

If a plugin has to rely on 4+ other dependencies, take a breath
and re-think either what you're trying to do.
Or at least look at what it's using all of those dependencies for.

7 - Dead code

every time you check in commented-out code, an orphan dies

This goes back to "not knowing your version control system"

Use a single commit removing whatever's been commented out

8 - Writing Testable Code

Structe applications around an object

IFFE's and anonymous doc-ready callbacks can make testing hard

9 - Don't Re-test jQuery

.html() and .css() have tests already

presumably so does everything else ;)

10 - No build step?

Do not pass go, do not collect $200

How many people are using some type of build process?

minification has its limits

ship the code that changes

11 - Bonus: Always use the latest jQuery

and if you have to use the compatibility plugin for 1.9, you have an opportunity to refactor

11.5 - Double Bonus Grab Bag

  • Multiple doc readys? The document is definitely ready.
  • Modifying natives. Don't.
  • UA sniffing. Stop it.
  • $.ajax({ async: false }); // wat
  • One less jpg

None of this is dogma

Have fun.

And keep breaking shit
