Stopping criminal behavior

So you don't end up like him...

Everything is going great.

Until it all goes to hell.

Things that will definitely, eventually happen:

  • Learn new code stuff
  • Adopt new libraries
  • Lose & gain team members
  • Probably be responsible for more than just JavaScript

Which means breaking shit
A lot.

Broken Windows: The outward appearance of criminal behavior can lead to actual criminality

If you fix broken windows you can fight software entropy

Even better, stop windows from breaking in the first place

Think less.

What constitutes crime in a software project?

  • when it effects producivity
  • or creates difficult bugs
  • or boxes you in
  • or confuses you later

Like any criminal justice system, there are minor crimes

Shoplifting (mixed tabs & spaces)

And major offenses

Regicide (Modifying natives Object.prototype.stopIt)

If you disagree, good! At least you're thinking critically

6

Leave $.fn alone unless you really mean it.

Questions you should axe before making a plugin

  • Does it use the DOM?
  • Is it generic enough for other people to use?
  • Are you going to release and maintain it?
6.5

Prefer simple dependencies over complex ones

If a plugin has to rely on 4+ other dependencies, take a breath
and re-think either what you're trying to do.
Or at least look at what it's using all of those dependencies for.

7

Dead code

Every time you check in commented-out code, an orphan dies

This goes back to "knowing your version control system"

Use a single commit removing whatever's been commented out

8

Writing Testable Code

Structure applications around an object

IFFE's and anonymous doc-ready callbacks can make testing hard

8.5

Don't Re-test jQuery

.html() and .css() have tests already

presumably so does everything else ;)

9

Don't Abuse Globals

Implicit Globals (bad)

var MyStuff = "is global :(";

Explicit Globals (not so bad)

window.MyStuff = "a bit better?";

Lots of explicit Globals (bad again)

window.App = {...};
window.Models = {...};
window.Collections = {...};
window.Config = {...};

Global variables destroy design information.

— Michael Feathers
